# 설명서 (비개발자용)

> 한국어 사용자를 위한 단계별 안내. 영문 [README](README.md) / [INSTALL.md](INSTALL.md)의 친절한 버전입니다.

---

## 1. 이게 뭔가요?

Hames는 **AI 도구 4종(Claude / Cursor / Codex / Gemini)을 한 세트로 묶어 쓰는 운영 규칙 모음**입니다.

- 각 AI 클라이언트에 흩어져 있던 룰을 한 곳에서 관리
- 작업 영역(워크스페이스)별로 컨텍스트 격리
- 위험한 명령은 자동 차단 (4중 방어선)

설치는 AI한테 한 줄 던지면 끝납니다. 실제 코드 작성은 안 합니다.

---

## 2. 누구한테 맞나?

### 맞는 사람
- AI 클라이언트를 **2개 이상** 동시에 쓰고 있음 (Claude Code + Cursor 등)
- AI한테 같은 룰을 매번 다시 설명하는 게 지겨움
- "이거 어디 폴더에서 작업했더라" 자주 헤맴
- AI가 의도치 않게 파일을 덮어쓰는 사고가 한 번이라도 있었음

### 안 맞는 사람
- AI 클라이언트 1개만 쓰고 충분히 만족함
- ChatGPT 웹만 쓰고 데스크톱 CLI 도구는 안 씀
- 룰 파일 만드는 게 귀찮음 (이건 룰 파일 운영 도구)

**판단 기준 한 줄:** "AI 4개를 동시에 굴리는데 룰이 제각각이라 짜증난 적 있다" → 맞음. 없으면 굳이.

## 3. 설치 전 준비물

### 3-1. 컴퓨터에 설치되어 있어야 하는 것 3개

세 가지 도구가 미리 깔려 있어야 합니다. 이미 있으면 넘어가도 됩니다.

| 도구 | 확인 방법 (PowerShell 또는 터미널에서) | 없으면 다운로드 |
|---|---|---|
| **Node.js 18 이상** | `node --version` → `v18.x.x` 이상 나오면 OK | https://nodejs.org/ (LTS 버전) |
| **Python 3.10 이상** | `python --version` → `3.10.x` 이상 나오면 OK | https://www.python.org/downloads/ |
| **Git 2.30 이상** | `git --version` → `2.30.x` 이상 나오면 OK | https://git-scm.com/downloads |

#### 확인 예시 (Windows PowerShell)

```powershell
node --version
python --version
git --version
```

세 줄 모두 버전 번호가 나오면 통과. "command not found" 또는 "인식되지 않습니다"가 나오면 그 도구를 설치하세요.

> 💡 **설치하기 너무 막막하면:** 유튜브에 "Node.js 설치 Windows", "Python 설치 Windows", "Git 설치 Windows" 검색하면 5분짜리 영상 많이 있습니다.

### 3-2. AI 클라이언트 1개

다음 중 **하나만** 깔려 있으면 됩니다 (여러 개 깔려 있으면 더 좋음):

- **Claude Code** (가장 추천) — https://claude.com/claude-code
- **Cursor** — https://cursor.com/
- **Codex CLI** — https://github.com/openai/codex
- **Gemini CLI** — https://github.com/google-gemini/gemini-cli

> ⚠️ **ChatGPT 웹/Claude.ai 웹은 안 됩니다.** 이 도구는 컴퓨터의 파일을 직접 만져야 해서 데스크톱/CLI 형태의 AI 클라이언트가 필요합니다.

## 4. 설치 (3분)

### 4-1. Hames를 설치할 폴더 정하기

새 폴더 하나를 정합니다. 예시:

- Windows: `C:\Users\본인이름\hames`
- Mac/Linux: `~/hames`

이 폴더가 이미 있을 필요는 없습니다. AI가 알아서 만듭니다.

### 4-2. AI 클라이언트 열기

설치한 AI 클라이언트(예: Claude Code)를 위에서 정한 폴더의 **상위 폴더**에서 엽니다.

예: `C:\Users\본인이름`에서 Claude Code 실행 → AI가 그 안에 `hames` 폴더를 만들어 설치합니다.

### 4-3. AI에게 한 줄 던지기

채팅창에 이거 한 줄만 복사해서 붙여넣으세요:

```
https://github.com/baek-labs/hames 의 SETUP_PROMPT.md 읽고 셋업해줘
```

### 4-4. AI가 묻는 것에 답하기

설치 도중 AI가 두 가지를 물어봅니다:

1. **설치 위치** — 위에서 정한 폴더 경로 답하기 (예: `C:\Users\본인이름\hames`)
2. **이름과 이메일** — 본인 이름과 이메일. 이건 룰 파일에 본인 이름이 들어가는 용도. 가짜로 적어도 작동에는 문제 없음.

### 4-5. 끝

3분쯤 후 AI가 "Hames install complete." 라고 말하면 성공.

## 5. 작동 확인 (3분 테스트)

### 5-1. 첫 메시지에 "방어선" 활성 확인

설치된 폴더로 들어가서 AI 클라이언트를 다시 엽니다 (예: Claude Code에서 `cd hames` 후 재시작 또는 그 폴더에서 새로 열기).

아무 메시지나 보냅니다 (예: "안녕"). AI 응답 안에 다음 두 줄이 보여야 합니다:

```
Loaded: CLAUDE.md, prompt_engineering.md, ...
Signatures: HAMES SYSTEM KERNEL v5.5 | DEEP_TASK_PROTOCOL | ...
```

이 두 줄이 보이면 시스템이 제대로 활성화된 것입니다.

### 5-2. `/doctor` 시스템 점검

채팅창에 `/doctor` 라고 입력합니다. AI가 시스템 무결성 점검을 실행하고 결과를 보고합니다. 빨간색 경고가 없으면 정상.

### 5-3. 처음 가볼 곳

- `docs/01_philosophy.md` — Hames가 왜 만들어졌는지 (영문)
- `HamesSystem_Public.md` — 시스템 전체 그림 (영문)
- `workspaces/_scaffold/` — 본인 워크스페이스 만들 때 복사할 템플릿

## 6. 첫 사용

설치만 하고 끝나면 의미 없습니다. 본인 작업 영역(워크스페이스)을 하나 만들어 보세요.

### 가장 단순한 첫 워크스페이스 만들기

```powershell
# Windows
Copy-Item -Recurse workspaces\_scaffold workspaces\my_first_workspace
```

```bash
# Mac/Linux
cp -r workspaces/_scaffold workspaces/my_first_workspace
```

그 후 AI한테:

```
workspaces/my_first_workspace/CLAUDE.md 를 읽고, 내가 [원하는 작업 종류]에 쓸 워크스페이스로 채워줘
```

AI가 템플릿을 본인 용도에 맞게 채웁니다.

## 7. 자주 묻는 질문

### Q1. 설치 도중 "npm install" 어쩌고 에러가 나요

대부분의 경우 **설치 자체에는 문제 없음**. arsenal 폴더의 추가 도구는 선택 사항이라, npm 에러가 나도 코어 시스템은 작동합니다. AI가 "Hames install complete." 라고 마지막에 말하면 OK.

### Q2. AI가 첫 메시지에서 "Loaded:" 줄을 안 보여줘요

AI 클라이언트를 그 폴더(`hames`)로 들어가서 **다시 시작**해야 합니다. 클라이언트는 시작할 때 그 폴더의 룰 파일을 읽기 때문에, 폴더 밖에서 열려 있던 세션은 룰을 모릅니다.

### Q3. 한국어로 안 됩니까? 다 영어인데

룰 파일과 문서는 영문입니다. 다만 AI한테는 한국어로 말 걸어도 됩니다. 룰은 AI가 읽고 이해하는 용도지, 사용자가 직접 읽을 필요는 없습니다.

### Q4. 4개 AI 클라이언트 다 설치해야 하나요?

아니요. 1개만 있어도 됩니다. 다만 Hames의 진짜 가치는 여러 클라이언트를 동시에 쓸 때 같은 룰이 자동 적용되는 것이라, 1개만 쓸 거면 효용이 줄어듭니다.

### Q5. 망쳤어요. 처음부터 다시 하고 싶어요

Hames 설치 폴더를 통째로 지우고 4번부터 다시 하면 됩니다. 다른 시스템에는 영향 안 줍니다 (init 스크립트가 본인 컴퓨터의 다른 파일은 안 건드립니다).

### Q6. 회사 컴퓨터에 깔아도 되나요?

MIT 라이센스라 법적으론 OK. 다만 회사 IT 정책에 따라 npm install / git clone이 막혀 있을 수 있으니 확인하세요. 회사 코드를 Hames 워크스페이스에 복사하는 건 회사 정책 따져야 합니다.

### Q7. 업데이트는 어떻게 하나요?

이 프로젝트는 **frozen reference**입니다. 추후 업데이트 안 합니다. 본인이 마음에 드는 부분만 가져다 fork해서 쓰는 게 정식 사용법.

## 8. 안 되면 (트러블슈팅)

### 설치 자체가 실패할 때
1. 3-1의 준비물 3개 버전 다시 확인
2. 인터넷 연결 확인 (git clone, npm install 모두 인터넷 필요)
3. 폴더 경로에 한글이 있으면 영문 경로로 바꿔서 시도

### 설치는 됐는데 작동이 이상할 때
1. AI 클라이언트를 hames 폴더 안에서 **다시** 시작
2. 첫 메시지 응답에 "Loaded:" / "Signatures:" 두 줄 나오는지 확인
3. `/doctor` 실행해서 어디가 깨졌는지 확인

### 그래도 안 될 때
- [INSTALL.md](INSTALL.md) — 영문 상세 가이드 (트러블슈팅 섹션 포함)
- [GitHub Issues](https://github.com/baek-labs/hames/issues) — 다만 frozen project라 답변 보장 안 됨
